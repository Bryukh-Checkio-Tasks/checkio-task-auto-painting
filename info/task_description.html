Nicola has built a semi-automatized painting system. But this system can paint only one side of an item, after it an operator must reload the item and start painting the other side (the system detects sides automatically). The painting process always takes the same amount of time. The camera can paint K surfaces at a time. Nicola wants Stephan to operate the painting system. That's why he should develop an algorithm for Stephan, that allows him to paint N (0 < N < 10) surfaces in the shortest possible timeframe. Be careful and don't load the item more than two times.

The items have numbers from 0 to 9. You are given capacity of the painting system (K) and quantity of items (N). You should return the sequence of painting as a string, where each action is the numbers of the painted items. Actions are separated by comma (",").

Input: Capacity of the painting system (K) and quantity of items (N). Integers.

Output: The sequence of actions, a string.

Example: 

checkio(2, 3)  # "01,12,02"
checkio(6, 3)  # "012,012"
checkio(3, 6)  # "012,012,345,345"
checkio(1, 4)  # "0,0,1,1,2,2,3,3"
checkio(2, 5)  # "01,01,23,42,34"

How it is used: This is a simple example of how to use graph searching for the optimization problem.
