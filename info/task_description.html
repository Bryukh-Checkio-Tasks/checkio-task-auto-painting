Nicola have built the semi-automatized painting system. But this system can paint only one side of the detail, after it an operator must reload the detail and start painting for the other side (the system detects sides automatically). The painting process always ahs the same time. The camera can paint K details at one time. Nicola wants to use Stephan for serving of the painting system. For it he should develop the algorithm for Stephan, that allows to paint N (0 < N < 10) details for the shortest time. Be careful and don't load the detail more than two times.

The details have numbers from 0 to 9. You are given capacity of the painting system (K) and quantity of details (N). You should return the sequence of painting as a string, where each action is the numbers of the painted details. Actions are separated by comma (",").

Input: Capacity of the painting system (K) and quantity of details (N). Integers.

Output: The sequence of actions, a string.

Example: 

checkio(2, 3)  # "01,12,02"
checkio(6, 3)  # "012,012"
checkio(3, 6)  # "012,012,345,345"
checkio(1, 4)  # "0,0,1,1,2,2,3,3"
checkio(2, 5)  # "01,01,23,42,34"

How it is used: This is a simple example of optimization program with graph searching.
